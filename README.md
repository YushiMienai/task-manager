# Добро пожаловать
 
Это простое одностраничное приложение (SPA) для управления задачами. Описание будет позже
Цель:
Создать простое одностраничное приложение (SPA) для управления задачами — добавление, отображение, изменение статуса и удаление.

Функциональность:
Список задач
Отображение всех задач.
Каждая задача: title, description, status (pending / in_progress / done).
Добавление новой задачи
Форма с title и description.
Изменение статуса задачи
Переключение между статусами.
Удаление задачи

Технические требования:
Стек: React
Стилизация: CSS / SCSS или TailwindCSS (на выбор).
Без использования UI-библиотек (напр. Material UI) — ручная верстка.
Хранение данных:
Можно использовать localStorage.
Или подключиться к REST API (если backend есть).
Кодовая структура: понятная, с разделением компонентов.
Адаптивность: хотя бы базовая (минимум — на экранах 320–1440px).
Бонусные (необязательные) фичи:
Фильтрация задач по статусу.
Анимации (например, при появлении задач).
Сохранение в localStorage.

Цель:
Разработать простой REST API сервис для управления списком задач. Каждая задача должна иметь заголовок, описание, статус и дату создания.

Требования к функционалу:
Создание задачи
Поля: title (string), description (string, optional), status (enum: pending/in_progress/done).
Дата создания проставляется автоматически.
Просмотр списка задач
Возможность фильтровать по status.
Обновление задачи
Можно изменить title, description, status.
Удаление задачи

Технические требования:
Язык: Python
Фреймворк: fastapi/django + react
Структура проекта: Чистая, логичная (желательно разнесённая по папкам).
Документация: Использовать Swagger/OpenAPI (если фреймворк поддерживает).

Бонусные (необязательные) пункты:
Написать хотя бы 2–3 unit-теста (например, на создание и получение задачи).
Использование миграций (Alembic / Prisma Migrate).
Докеризация (Dockerfile).
Авторизация (например, простая токен-авторизация).
